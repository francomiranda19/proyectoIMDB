Dataset¶

Se trabaj con el dataset de IMDB, el cual est distribuido en 7 tablas en formato TSV, de las cuales se ocuparn cinco.

La tabla title_basics contiene varios atributos de inters sobre ttulos:

  • titleType: tipo/formato del ttulo (pelcula, cortometraje, documental, etc.)

  • primaryTitle: el nombre ms popular del ttulo

  • startYear: ao de lanzamiento original

  • runtimeMinutes: duracin en minutos

  • genres: arreglo de hasta tres gneros asociados al ttulo

La tabla title_ratings contiene dos atributos interesantes:

  • averageRating: promedio de todas las valoraciones de los usuarios de IMDB que ha recibido el ttulo

  • numVotes: nmero de valoraciones que el ttulo ha recibido

La tabla title_crew contiene dos atributos de inters:

  • directors: id de los directores

  • writers: id de los escritores

La tabla title_principals tiene cinco atributos relevantes:

  • ordering: nmero para identificar filas para un mismo titleId

  • nconst: identificador nico de la persona

  • category: la categora del trabajo de la persona

  • job: trabajo especfico de la persona

  • characters: el nombre del personaje que se actu, si es aplicable a la persona

La tabla name_basics contiene cinco atributos importantes:

  • primaryName: nombre de la persona

  • birthYear: fecha de nacimiento

  • deathYear: fecha de muerte, si es aplicable

  • primaryProfession: top 3 profesiones de la persona

  • knownForTitles: ttulos por los que la persona es conocida

Reformulación de preguntas y problemas¶

Luego de haber agregado las columnas correspondientes a directores, escritores y actores para cada una de las pelculas, ahora se reformularn las preguntas y problemas. Para estas preguntas se defini xito como la valoracin y popularidad como la cantidad de votos.

  • Es posible predecir el xito de una pelcula en base a su director y su gnero?

  • Existe una relacin entre el gnero y los clusters formados a partir de su ao y duracin?

  • Es posible predecir la popularidad de una pelcula en base a la presencia de un actor en especfico?

Propuesta experimental¶

  • Para responder la primera pregunta se usarn distintos mtodos de clasificacin. En primer lugar, se etiquetarn los datos que se necesitan para entrenar los clasificadores. Se etiquetar el xito con 4 etiquetas, con averageRating sobre 7.5 se etiqueta con 3, entre 5 y 7.5 con 2, entre 3.5 y 5 con 1 y bajo 3.5 con 0. Luego, se debe etiquetar cada director con un id nico y repetir las instancias de pelculas con ms de un director. Esto mismo se tiene que repetir para los gneros. Con esto se entrenarn los clasificadores usando un clasificador Dummy, rbol de Decisin, KNN, Naive-Bayes, y luego se compararan los resultados entre estos usando las mtricas de precision, recall y F1-Score. Si alguna de estas mtricas es mayor al 80% entonces se dir que s es posible la prediccin.
  • Para responder la pregunta 2 se usarn mtodos de clasificacin no supervisada como K-Means, Clustering Jerrquico Aglomerativo y DBSCAN para crear los clusters. Se probarn todos estos mtodos con el ao, la duracin de la pelcula, y tambin con cada atributo por separado para comparar cmo se generan mejor los clusters. Se evaluarn todos los clusters que se formaron usando mtricas de matriz de incidencia, SSE, cohesin y separacin. Finalmente se calcular la presencia relativa (i.e porcentaje de datos que pertenecen a un gnero) de los gneros en cada cluster y se ver si existen gneros dominantes en cada uno. Si no se evidencian gneros dominantes, significa que no existe (o no encontramos) una relacin entre el gnero y los clusters formados a partir de su ao y duracin.
  • Para responder la tercera pregunta nuevamente se usar clasificacin supervisada. Para esto en primer lugar se deben etiquetar los datos. Para etiquetar la popularidad se seguir un mtodo similar al etiquetado del xito, dividiendo la cantidad de votos en 4 intervalos y clasificando cada intervalo del 0 al 3. Luego se debe clasificar la presencia de un actor en especfico con 0 si el actor no est en la pelcula y 1 en el caso contrario. Para esto se crear una funcin que reciba el actor que se quiere usar para la clasificacin de los datos y el ao en el que empez (y termin si es que aplica) su carrera como actor. Esta funcin etiquetara con cero todos las pelculas que no entren en este intervalo y para el resto de los datos recorrer los actores y etiquetara con 1 si esta presente. Esto se probara con distintos actores para hacer distintas clasificaciones. Luego, con los datos etiquetados se entrenarn los diferentes clasificadores usando un clasificador Dummy, rbol de Decisin, KNN y Naive-Bayes para luego evaluar los resultados con precision, recall y F1-Score.

Clasificación de datos¶

Pregunta 1¶

Luego de haber etiquetado los datos necesarios, se clasificarn los datos utilizando varios mtodos y calculando su precision, recall y F1-Score. Si alguno de estos valores entrega un valor mayor o igual a un 80% se dir que s se puede predecir.

La idea a seguir ser realizar un cierto nmero de tests con distintos training set, y el valor de precision, recall y F1-Score ser el promedio de todos los valores calculados.

In [47]:
from sklearn.dummy import DummyClassifier
from sklearn.tree import DecisionTreeClassifier
from sklearn.neighbors import KNeighborsClassifier
from sklearn.naive_bayes import GaussianNB
from sklearn.metrics import f1_score, recall_score, precision_score
from sklearn.model_selection import train_test_split

def run_classifier(clf, X, y, num_tests=10, test_size=.33):
    metrics = {"precision": [], "recall": [], "f1-score": []}
    for _ in range(num_tests):
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=test_size)
        clf.fit(X_train, y_train.values.ravel())
        predictions = clf.predict(X_test)
        metrics["y_pred"] = predictions
        metrics["precision"].append(precision_score(y_test, predictions, average='macro', zero_division=0))
        metrics["recall"].append(recall_score(y_test, predictions, average='macro'))
        metrics["f1-score"].append(f1_score(y_test, predictions, average='macro'))
    return metrics
